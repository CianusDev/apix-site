## Commands

```bash
pnpm dev          # Start development server
pnpm build        # Production build
pnpm lint         # ESLint check
pnpm type-check   # TypeScript type check (tsc --noEmit)
pnpm generate-keys  # Generate a SESSION_SECRET (openssl rand -base64 32)
```

Always run `pnpm type-check` and `pnpm lint` before finishing any task.

## Environment Setup

Copy `.env.example` to `.env.local` and fill in:

```env
SESSION_SECRET=<32+ char secret from pnpm generate-keys>
NEXT_PUBLIC_API_URL=http://localhost:3001/api
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_APP_NAME=MonApp
```

Environment variables are validated at startup via Zod in `src/config/env.ts`. Use `serverEnv` for sensitive server-side vars and `clientEnv` for `NEXT_PUBLIC_*` vars. Never access `process.env` directly elsewhere.

## Architecture

This is a **feature-based Next.js 16 App Router** project with strict Server Component conventions.

### Core Rule: Pages are always Server Components

- All `page.tsx` files must be `async` Server Components — no `"use client"`
- Exception: `error.tsx`, `not-found.tsx` require `"use client"` (Next.js requirement)
- All pages that fetch data must use `createLoader` from `@/lib/loader`

### createLoader Pattern (mandatory for SSR data fetching)

```typescript
import { createLoader } from "@/lib/loader";

const loadData = createLoader("/users/[id]", async ({ params, searchParams }) => {
  return await getUserById(params.id); // params.id is auto-typed from route
});

export default async function Page(props: typeof loadData.Props) {
  const data = await loadData(props);
  return <div>{/* ... */}</div>;
}
```

`createLoader` wraps `React.cache()` for deduplication and provides automatic TypeScript typing of route params via `@/routes` (generated by Next.js build).

### Feature-Based Structure

Each feature lives in `src/features/{feature}/` and is fully self-contained:

| File | Directive | Role |
|------|-----------|------|
| `{feature}.service.ts` | `"use server"` | Server Actions — API calls via `api` + `processApiData` |
| `use-{feature}.ts` | `"use client"` | Client hook — state, loading, error, refetch |
| `{feature}.guard.ts` | server-only | Route protection via `redirect()` |
| `components/` | per need | Feature-specific UI components |
| `{feature}.types.ts` | — | DTOs and local types (optional) |

### Data Flow

```
Page (Server Component)
  └─▶ createLoader ──▶ *.service.ts ("use server") ──▶ api (Axios) ──▶ Backend
                                                     ◀── processApiData()
  └─▶ Client Component
        └─▶ use-*.ts hook ──▶ *.service.ts (Server Action)
```

### Where to Put New Code

| What | Where |
|------|-------|
| New page | `src/app/{route}/page.tsx` |
| Business component | `src/features/{feature}/components/` |
| Shared UI (generic) | `src/components/ui/` (shadcn only, don't modify) |
| Shared layout | `src/components/layouts/` |
| Feature-specific hook | `src/features/{feature}/use-*.ts` |
| Global hook | `src/hooks/use-*.ts` |
| Feature service | `src/features/{feature}/*.service.ts` |
| Cross-feature service | `src/services/*.service.ts` |
| Business entity type | `src/models/*.model.ts` |
| Global store | `src/stores/*.store.ts` (Zustand) |
| React provider | `src/providers/` |
| Constants | `src/lib/constants.ts` |

### Key Config Files (modify carefully)

- `src/config/env.ts` — Zod-validated env vars, fails fast at startup
- `src/config/session.ts` — JWT session management (`encrypt`, `decrypt`, `createSession`, `verifySession`, `deleteSession`) — server-only
- `src/config/axios.ts` — Axios instance with auth token interceptor and 401 redirect; always import `api` and `processApiData` from here, never create a new Axios instance
- `src/config/types.ts` — Shared types: `SessionPayload`, `APIResponse`

### Service Pattern

```typescript
"use server";
import { api, processApiData } from "@/config/axios";
import { APIResponse } from "@/config/types";

const endpoint = "/users";

export async function getUsers(): Promise<APIResponse> {
  return processApiData(api.get(endpoint));
}
```

Never try/catch inside services — `processApiData` handles all errors and standardizes `{ success, data, message }`.

## Naming Conventions

| Type | File | Export |
|------|------|--------|
| Component | `kebab-case.tsx` | `PascalCase` (named export) |
| Hook | `use-*.ts` | `useFeatureName` |
| Service | `*.service.ts` | `camelCase` functions |
| Model | `*.model.ts` | `PascalCase` type/interface |
| Guard | `*.guard.ts` | `camelCase` functions |
| Constant | in `constants.ts` | `SCREAMING_SNAKE_CASE` |

- Never use `any` — use `unknown` or generics
- Never use `@ts-ignore` or `@ts-nocheck`
- Use named exports (not default) for components
